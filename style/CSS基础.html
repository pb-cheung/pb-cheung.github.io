# 文档流

  • 普通流（Normal Flow）
    • 元素按照其在HTML中的先后位置从上到下布局
    • 行内元素水平排列，直到当前行占满然后换行；块级元素则会被渲染为一个完整的新行
    • 所有元素默认都是普通流定位
  • 浮动流（Float）
    • 浮动的元素会脱离普通流，并吸附到父元素的左侧/右侧（根据自己设置），在正常布局中位于浮动元素之下的内容，此时会围绕着浮动元素，填满其右侧空间。
  • 绝对定位流（Absolute positioning）
    • 绝对定位的元素会脱离普通流，位置由top、left这些定位属性决定

# 浮动布局

清除浮动

浮动元素会脱离正常文档流，其他元素会显示在其下方。如果不想让剩余元素受到此影响，可以使用清除浮动。

# 外边距重叠

margin collapsing

上下两个块级元素，它们之间的外边距不是二者的外边距相加之和，而是较大的那个外边距，这种现象就是边距重叠。具体有三种情况：

# 1.相邻同级元素

<style>
p:nth-child(1){
  margin-bottom: 13px;
}
p:nth-child(2){
  margin-top: 87px;
}
</style>

<p>下边界范围会...</p>
<p>...会跟这个元素的上边界范围重叠。</p>

两个段落的间距是87px而不是87 + 13 = 100px。

解决方法：把其中一个或者两个都放入BFC容器中。

# 2.父子元素

没有其他约束的情况下，

  • 父元素与第一个元素的子元素上外边距
  • 父元素与最后一个子元素的下外边距
<style type="text/css">
    section    {
        margin-top: 13px;
        margin-bottom: 87px;
    }

    header {
        margin-top: 87px;
    }

    footer {
        margin-bottom: 13px;
    }
</style>

<section>
    <header>上边界重叠 87</header>
    <main></main>
    <footer>下边界重叠 87</footer>
</section>

消除方法：

  • 修改父元素属性（例如添加overflow:hidden;），创建BFC。

# 3. 空元素

空元素自身的上下边距重叠

<style>
* {
  margin, padding: 0;
}
p {
  margin: 0;
  background-color: #ccc;
}
div {
  margin-top: 13px;
  margin-bottom: 87px;
}
</style>

<section>
  <p>下边界范围是 87 ...</p>
  <div></div>
  <p>... 上边界范围是 87</p>
</section>

section 的高度为132px = 22.5px * 2 + 87px

解决方法：为空元素添加属性，让其变为BFC元素。

参考：外边距重叠|MDN (opens new window)

# BFC

块级格式化上下文 （Block Formating Context）

BFC是个独立的隔离的容器，容器里的子元素不会影响到外面元素，反之亦然，它还有一些普通块级元素没有的特性，BFC元素属于普通流。

# BFC渲染规则：

  1. BFC元素的子元素垂直方向上一个接一个的放置（普通流的特性）；
  2. 垂直方向上相邻的两个子元素会发生外边距重叠，折叠后的外边距取较大的值（普通流的特性）；
  3. BFC元素与普通块级元素、两个BFC元素之间不会不会发生边距重叠的现象（独有特性）；
  4. 子元素的左右外边距不会超过父BFC元素的的对应边界，即使子元素是浮动元素。

# 创建BFC的方式（脱离文档流）:

  1. 根元素（<html>）
  2. 使用浮动，float取值不为none，使用flex布局
  3. 使用绝对定位，position: absolute、fixed
  4. 行内块，display: inline-block
  5. overflow取visible之外的值

# BFC应用

# 1.解决外边距合并

请看上一节

# 2.清除浮动

容器内元素浮动，脱离了文档流，所以容器的高度只有上下边框2px。 容器上添加overflow:hidden;变为BFC，容器就会包裹浮动元素（浮动局限在BFC内）。

# 3.阻止浮动元素对元素的覆盖

浮动元素（蓝色）位于红色元素上部产生遮挡，将第二个红色元素变为BFC，则两个元素不再发生重叠。

# 百分比布局

不同元素的百分比基准不同。

  • 尺寸，相对于直接父元素。width、height
  • 边距
    • 相对于直接父元素的width，margin、padding
  • 定位
    • 相对于直接父元素的高度，top、bottom
    • 相对于直接父元素的宽度，left、right
  • 自身
    • 相对于自身宽度，border-radius、translate、background-size

缺点显而易见：

  1. 计算复杂，设计稿数值需要转换为百分比
  2. 基准不固定

# rem布局

rem——root em。以根元素的字体大小为基准 em——父元素字体的大小

实践中的操作： 默认浏览器中的文字是16px，那么就相当于1rem = 16px; 为了更方便的根据设计稿从px换算成rem，把html根元素的font-size写成62.5%，这样1rem=10px。

示例：

    html {
      font-size: 62.5%;/* 10➗16 = 0.625*/
    }
    /*定义一个高35px，宽46px的盒子*/
    .div{
        width: 3.5rem;
        height: 4.6rem;
    }

# 视口单位（vw/vh）

vw，viewPort width，视口宽度。1vw = 视口宽度的1%。 vh，viewPort height，视口高度。1vh = 视口高度的1%。

解决了百分比基准不统一的问题。

# SASS（预处理器）

# SASS解决的问题

CSS是一行行的描述语言，不是编程语言（没有变量，没有条件判断），不够灵活方便。

CSS预处理器（css preprocessor）就是为了解决这个问题而创造的，它用一种编程语言进行网页样式设计，然后再编译成CSS文件使用。

SASS是 CSS预处理器的一种。

# SASS语法

后缀：

  • SCSS。新版语法，.scss。是CSS3的超集。一般是使用这个。
  • SASS。旧版语法，后缀.sass。缩进语法——没有逗号和分号，依靠缩进来指定代码块。

语法：

  • 变量（$开头）

  • 计算

  • 嵌套（选择器嵌套、属性嵌套）

  • 注释（/**/和//）

  • 继承 （@extend .class1;）

  • Mixin（@mixin name 定义一个代码块，使用@include name调用复用）

  • 颜色函数

  • 条件判断语句（@if/@else）

  • 循环语句（@for/@while/@each）

  • 自定义函数（@function）

&连体符号，和伪元素、伪类配合使用，使用方法是用&替代元素自身。
