# 浏览器的回流和重绘

# Reflow-回流

# 概念

渲染树（Render Tree）的一部分发生改变（树上的节点发生的位置和尺寸发生改变） ，引起浏览器的重新渲染叫做reflow，或者叫做layout、layouting。

回流又名重排，指几何属性需改变的渲染。 可理解成将整个页面填白，对内容重新渲染一次。人眼并不能感受到这一过程。

# 引发回流的操作

  • 元素尺寸、位置变化
  • 窗口尺寸变化
  • 元素字体变化
  • 添加or删除可见的DOM
  • 激活伪类（例如：:hover）
  • 盒子相关的API调用（elem.clientWidth）
  • 滚动相关的API调用（ elem.scrollTo）

# Repaint-重绘

屏幕中的一部分（对应DOM树种的某个节点）的样式发生变化， 只引发该部分单独的显示效果变化叫做repaint。

重绘指更改外观属性而不影响几何属性的渲染。

# 引发重绘的操作

  • 修改background
  • 修改visiblility
  • 修改border

# 优化

回流和重绘对于浏览器来说是代价昂贵的，应当尽可能避免。

# 避免措施

  • 避免使用table布局
  • 避免设置多层内联样式
  • 避免频繁修改样式，最好一次性重写style属性，将样式列表定义为class，一次性修改class属性
  • 尽可能在DOM树的最末端（树的叶子）修改改变class
  • 避免使用CSS表达式（calc）
  • 有动画的元素将其脱离文档流（positioin: absolute|fixed），这样修改样式只引起重绘，不会造成回流
  • transform代替top
  • CSS3硬件加速（GPU加速）
