# 面向对象编程、原型及原型链

# 一、面向对象

# 1. 对象的理解

  • 对事物的抽象
  • 是一个容器（盛放了属性和方法）
    • 属性：对象的状态
    • 方法：对象的行为

# 2. 构造函数

作用是构造（成成）对象

构造函数（类）是生成对象的模板，表征了一类物体的特征。

JavaScript 系统是由构造函数和原型链组成的。

举例：

function Course(teacher) {
    this.teacher = teacher;
    this.leader = 'xy';
    this.startCourse = function(name) {
        return `开始${name}课程`;
    }
}
const course = new Course('yy'); //新的对象

解释：

  1. Course就是一个构造函数
  2. 函数体内的this指向函数生成的实例对象
  3. 需要通过new来实例化
  4. 可以做初始化传参（teacher）

不初始化直接调用不会生成对象实例，全局作用域下调用，函数内部this指向全局上下文（window）。

# 3. new运算符

new 构造函数生成实例对象是怎样一个过程：

  1. 创建一个空对象
  2. 绑定this到这个空对象
  3. 以1.创建的对象为上下文，执行构造函数（初始化）
  4. 返回1.创建的对象

# 4. constructor属性

在Chrome开发者工具中打印创建的实例，可以看到实例有一个constructor属性。

course.constructor === Course; // true

解释

  1. 每个实例化的对象都自动拥有constructor属性
  2. 这个属性继承自构造函数的原型（Course.prototype），指向了构造函数的引用

实例对象天然继承构造函数（类）中的属性和方法（course.teacher、course.startCourse等）。 但是这种方式是有缺点的，比如，方法会存在所有的实例对象中，造成空间浪费：

    const course1 = new Course('this');
    const course2 = new Course('OOP');
    // 构造函数中的方法（startCourse）会存在于每一个生成的实例内，
    // 而方法往往需要模板统一化，重复的挂载只会浪费资源

原型继承可以解决这个问题

function Course(teacher) {
    this.teacher = teacher;
    this.leader = 'xy';
}
// 方法放在原型链上，所有的实例可以共用
Course.prototype.startCourse = function(name) {
    return `开始${name}课程`;
}
const course1 = new Course('Math');
course1.getName(); // 'Math'
const course2 = new Course('Enslish');
course2.getName(); // 'English'

# 二、原型及原型链

# 原型对象

（构造函数的）原型对象是什么？

  1. 构造函数：用来创建对象初始化对象的函数
    • 构造函数中拥有一个prototype属性，和实例的__proto__指向相同的地址空间
  2. 实例对象：根据构造函数创建
    • 每个实例都有一个constructor属性，是由继承而来，指向构造函数

实例可以通过hasOwnProperty判断是否是自身属性，通过prototype继承的属性返回false。

# 三、继承

继承是类与类之间的关系，不是对象与对象

# 原型链继承

通过原型链把目标对象与其父子串联起来，当访问某一属性时，可以顺着原型链查找，达到继承的效果。

function Game() {
    this.name = 'LOL';
    this.color = ['red'];
}
Game.prototype.getName = function() {
    return this.name;
}
// 子类
function LOL() {}
LOL.prototype = new Game(); // 子类原型指向父类实例
LOL.prototype.constructor = LOL;
// 实例
const lol1 = new LOL();
const lol2 = new LOL();
console.log(lol1 instanceof Game); // true
console.log(lol1 instanceof LOL); // true
// 子类实例共享父类实例的属性
lol1.color.push('black');
console.log(lol1.color); // ['red', 'black']
console.log(lol2.color); // ['red', 'black']

这种方式实现了继承，但是还有两点问题：

  1. 在创建实例的时候，不能向父类的构造函数中传参（new LOL(pram) param并不会传到父类构造函数中）
  2. 父类实例属性给了子类的原型，这些属性就会被所有子类属性共享，会造成会乱

# 构造函数继承（经典继承）

在子类构造函数中调用父类构造函数。

function Game() {
    this.name = 'LOL';
    this.color = ['red'];
}
Game.prototype.getName = function() {
    return this.name;
}
function LOL(param) {
    Game.call(this, param); // 子类构造函数中调用父类构造函数
}
const lol1 = new LOL('myid');
const lol2 = new LOL('myid2');
console.log(lol1.getName()); // 报错："TypeError: lol1.getName is not a function"
// 子类实例使用父类属性互不影响
lol1.color.push('black');
console.log(lol1.color); // ['red']
console.log(lol2.color); // ['red','black']

父类的属性可以被继承，问题：父类原型上的属性不会被继承。

# 组合继承

结合原型链继承和构造函数继承。 创建子类实例时候（new LOL('abc')）传参会被父类构造函数执行； 每次创建实例的时候，都会调用父类构造函数，子类实例不会共享同一个父类实例属性

function Game() {
    this.name = 'LOL';
    this.color = ['red'];
}
Game.prototype.getName = function() {
    return this.name;
}
function LOL(param) {
    Game.call(this, param); // 子类构造函数中调用父类构造函数
}
LOL.prototype = new Game();
LOL.prototype.constructor = LOL;

仍然存在一个问题：父类构造函数被调用两次，浪费空间。创建子类实例时被调用一次，原型链设置又调用一次。

# 寄生组合继承

在组合继承的基础上优化，使用Object.create(Game.prototype)创建一个新的原型对象赋予子类， 从而解决组合继承的缺陷。

function Game() {
    this.name = 'LOL';
    this.color = ['red'];
}
Game.prototype.getName = function() {
    return this.name;
}
function LOL(param) {
    Game.call(this, param);
}
LOL.prototype = Object.create(Game.prototype);
LOL.prototype.constructor = LOL;

# 多重继承

使用Object.assign合并两个父类的原型对象。

function Game() {
    this.name = 'LOL';
    this.color = ['red'];
}
Game.prototype.getName = function() {
    return this.name;
}
// 父类1
function Store() {
    this.shop = 'steam';
}
Store.prototype.getPlatform = function() {
    return this.shop;
}
// 父类2
function LOL(param) {
    Game.call(this, param);
}
LOL.prototype = Object.create(Game.prototype);
Object.assign(Store.prototype, LOL.prototype); // 疑问，只是继承了Store的属性。
LOL.prototype.constructor = LOL;

const lol1 = new LOL();
console.log(lol1 instanceof Store); // false
