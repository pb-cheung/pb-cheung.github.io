# keep-alive

  • <keep-alive>是一个内置组件之一，主要用于保留组件状态或避免重新渲染。
  • 它是一个抽象组件：它自身不会渲染一个 DOM元素，也不会出现在组件的父组件链中（$children/$parent）。
  • 和<component>或router-view搭配使用

# 组件源码

<keep-alive>内置组件的定义在src/core/components/keep-alive。它定义了自己的render函数。

  • render函数的作用是获取组件vnode，这里额外添加了vnode缓存功能（以及黑白名单控制）
  • getFirstComponentChild函数，即只获取处理它的第一个子组件

render函数源码：

  render() {
    const slot = this.$slots.default
    const vnode = getFirstComponentChild(slot)
    const componentOptions = vnode && vnode.componentOptions
    if (componentOptions) {
      // check pattern
      const name = _getComponentName(componentOptions)
      const { include, exclude } = this
      if (
        // not included
        (include && (!name || !matches(include, name))) ||
        // excluded
        (exclude && name && matches(exclude, name))
      ) {
        return vnode
      }

      const { cache, keys } = this
      const key =
        vnode.key == null
          ? // same constructor may get registered as different local components
            // so cid alone is not enough (#3269)
            componentOptions.Ctor.cid +
            (componentOptions.tag ? `::${componentOptions.tag}` : '')
          : vnode.key
      if (cache[key]) {
        vnode.componentInstance = cache[key].componentInstance
        // make current key freshest
        remove(keys, key)
        keys.push(key)
      } else {
        // delay setting the cache until update
        this.vnodeToCache = vnode
        this.keyToCache = key
      }

      // @ts-expect-error can vnode.data can be undefined
      vnode.data.keepAlive = true
    }
    return vnode || (slot && slot[0])
  }

# 渲染过程

示例：

<body>
    <div id="app">
        <keep-alive>
            <component :is="component"></component>
        </keep-alive>
        <button @click="component='my-a'">切换为组件A</button>
        <button @click="component='my-b'">切换为组件B</button>
    </div>
    <script>
        new Vue({
            el: "#app",
            data() {
                return {
                    component: 'my-a'
                }
            },
            components: {
                'my-a': {
                    template: '<div>my-a comp</div>'
                },
                'my-b': {
                    template: '<div>my-b comp</div>'
                },
            }
        })
    </script>
</body>

# 首次

  • <keep-alive>子组件首次渲染和普通组件一样，创建实例 => $mount => render
  • 自定义render中获取第一个子元素的vnode
  • 调用patch创建vnode对应真实DOM

# 缓存渲染

点击按钮切换组件

  • patch => patchVnode => updateChildren => patchVnode传入<keep-alive>组件新旧vnode => prepatch => updateChildComponent
  • <keep-alive>存在子元素，则调用该组件的watcher更新 => updateComponent => _update => render
  • 自定义render中，从缓存获取vnode并返回到_upate => patch => createElm => createComponent => init
  • init这里和首次渲染进入不同逻辑分支，已有了实例vnode.componentInstance，不用再创建和挂载，执行完毕返回到createComponent
  • createComponent中调用insert从vnode.elm获取真实dom插入父组件中
function updateChildComponent() {
  // resolve slots + force update if has children
  if (needsForceUpdate) {
    vm.$slots = resolveSlots(renderChildren, parentVnode.context)
    vm.$forceUpdate()
  }
}

const componentVNodeHooks = {
  init(vnode: VNodeWithData, hydrating: boolean): boolean | void {
    if (
      vnode.componentInstance &&
      !vnode.componentInstance._isDestroyed &&
      vnode.data.keepAlive
    ) {
      // kept-alive components, treat as a patch
      const mountedNode: any = vnode // work around flow
      componentVNodeHooks.prepatch(mountedNode, mountedNode)
    } else {
      const child = (vnode.componentInstance = createComponentInstanceForVnode(
        vnode,
        activeInstance
      ))
      child.$mount(hydrating ? vnode.elm : undefined, hydrating)
    }
  },
}
