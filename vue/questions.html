# Vue面试题

# 1.vue.$set原理是什么？

# 背景

this.data.arr[1] = 'bbb';
this.data.obj['xyz'] = 'xxx';

上面这种，常规赋值方式，修改某个数组元素，为对象增加属性在Vue.js中不能被响应更新视图。

原因：

  1. 数组的非对象元素没有被添加响应式（即arr[0]没有对应的getter和setter）；
  2. Vue.js中实现响应式是使用Object.defineProperty这个API，这个API无法检测到目标对象属性的新增和删除。

# 用法：

  1. data中的数组使用索引修改值
  2. data中对象属性添加成员
Vue.set(target, indexNum, value); // 修改数组某个索引的值
Vue.set(target, key, value); // 修改对象属性值
  • target为数组，调用Vue中重新封装的splice方法。
  • target为对象
    • target中已有key属性，直接修改（已经是响应式了）
    • target不是响应式（通过__ob__属性判断）对象，直接添加key属性
    • target是响应式对象，且没有key属性，定义响应式并通知。defineReactive(ob.value, key, value) + ob.dep.notify()

# 2.v-for/v-if/v-model的原理

# v-if/v-for

它们都是指令。 出现在模板中的标签上，模板会被编译成render函数。

let compiler = require('vue-template-compiler');
const ast = compiler.compile('<div v-if="false"></div>');
console.log(ast.render); // with(this) {return (false)?_c('div'):_e()}

const ast2 = compiler.compile('<div v-for="i in 3"></div>');
console.log(ast2.render); // with(this) {return _l((3), function(i){return _c('div')}) }

从以上demo可以看出

  • v-if被编译成了三元表达式。（_c创建元素，_e创建空元素）
  • v-for被编译成了一个函数。（_l对应renderList函数）
  • v-model
    • 表单元素上，实现双向绑定。不同类型的表单生成不同的render代码（指令 + value（innerHtml） + input/change等触发表单值改变的事件处理函数）。

# 3.v-model原理

# 背景

Vue的核心特性：数据响应式和数据双向绑定。

数据双向绑定就是通过v-model指令实现的。

# 用法

在表单中：

在表单元素（<input>、<textarea>和<select>等）上使用v-model指令，会创建双向数据绑定。 它会监听用户的输入，然后根据控件类型自动选取正确的方法来更新元素。 v-model指定的Vue实例属性是表单的初始值，用户编辑表单后该属性变量也会同步改变。

# 原理

v-model实际上是语法糖。Vue根据表单类型，将v-model转换添加对应的取值（domProps）表达式和变化响应事件（on）。

# 4.Vue中观察者模式和发布订阅模式的区别和场景

  • 响应式数据变化，是典型的观察者模式
  • 事件绑定是发布订阅模式

观察者（watcher）会被"被观察者"（属性dep）收集，数据变化的时候，被观察者会通知所有的观察者。 也可以表述为：订阅者（watcher）订阅属性变化事件，属性变化的时候，发布这个信息，订阅者收到。

观察者模式包括了发布订阅模式。

观察者模式中，观察者和被观察者是存在关联的。发布订阅模式中，订阅者和发布者没有关联（可以只发布不订阅）。
