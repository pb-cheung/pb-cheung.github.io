# 响应式原理

# 什么是响应式（Reactive）？

当数据变化时，相关的视图会自动更新，不需要手动操作。

# Object.defineProperty

通过Object.definePropertyAPI实现，它可以修改参数对象成员的get和set属性（以及其他属性例如writeable这里不关心）。目标对象成员被读取和修改分别会触发get方法和set方法，我们就可以在重写的这两个属性方法中做文章来实现响应式。

# 响应式添加

  1. 初始化过程中调用添加响应式函数：_init => initState => initData => observe

  2. 通过__ob__判断是否已经添加响应，未添加过则创建Observer实例，defineReactive

  3. Observer构造函数中，判断参数是否是数组

    • 非数组则遍历参数对象，调用defineReactive为其所有属性添加响应式
    • 如果是数组，遍历数组成员，调用observe为每一个数组元素添加响应式，然后还要对数组本身添加响应式（下一节说明）

# 数组响应式的实现

Object.defineProperty只能劫持已经存在的属性，后增的或者删除的不知道，因此，通过像Array.prototype.push()这样的方法为数组元素的操作并不能感知到，数组的响应式要特殊处理。

# 方法一：Vue2会为此单独写一些API(vue.$set/ vue.$delete)

# 方法二：劫持+修改数组方法

修改数组原型链:

data.__proto__ = newArrayProto; // data为数组

AOP修改会造成数组元素增删的方法，运行原生方法，对新增元素添加响应式。

let originArrayProto = Array.prototype;
let newArrayProto = Object.create(originArrayProto);
['push','shift'].forEach(method => {
    this.observeArray(data); // 为新添加的元素 添加响应式
    return originArrayProto[method]
})

完整代码：

class Observer {
    constructor(data) {
        this.dep = new Dep();
        // 作用：1.Observer实例，改写的数组方法被调用时候使用；2.标识该属性已被观测过
        Object.defineProperty(data, '__ob__', {
            value: this,
            enumerable: false // 将__ob__变成不可枚举，循环的时候遍历不到，避免无限循环爆栈
        })
        // data.__ob__ = this;
        // Object.defineProperty只能劫持已经存在的属性，后增的或者删除的不知道
        // Vue2会为此单独写一些API(vue.$set/ $delete)
        if(Array.isArray(data)) {
            // 重写数组方法
            data.__proto__ = newArrayProto;
            this.observeArray(data);
        } else {
            this.walk(data);
        }
    }
    walk(data) {
        // "重新定义"属性，（性能差）
        Object.keys(data).forEach(key => defineReactive(data, key, data[key]));
    }
    observeArray(data) {
        data.forEach(item => observe(item));
    }
}


let originArrayProto = Array.prototype;

let newArrayProto = Object.create(originArrayProto);

let methods = [ // 找到所有的变异方法（会改变数组）
    'push',
    'pop',
    'shift',
    'unshift',
    'reverse',
    'sort',
    'splice'
];

methods.forEach(method => {
    newArrayProto[method] = function (...args) {
        const result = originArrayProto[method].call(this, ...args); // 内部调用原来的方法，函数的劫持，切片编程
        let inserted;
        switch (methods) {
            case 'push':
            case 'unshift':
                inserted = args;
                break;
            case 'splice':
                inserted = args.slice(2);
                break;
            default:
                ;
        }
        let ob = this.__ob__;
        if (inserted) {
            // 对新增内容再进行观测
            ob.observeArray(inserted);
        }

        console.log('更新');
        ob.dep.notify(); // 数组变化了，通知对应的watcher实现更新逻辑
        return result;
    }
})

export {newArrayProto};

# 依赖管理

# 什么是依赖

通过前文，我们已经知道了解了是响应式、实现响应式的底层API、如何给数据添加响应式等。

数据的变化我们能感知到（通过setter），那么如何才能把这个变化更新到视图呢？ 视图和数据就是一组依赖关系。

Vue2中在这里使用了发布-订阅的设计模式，引入了三个角色Watcher、Dep和Observer。

  • Watcher顾名思义是观察者or订阅者，可以是渲染视图、computed、watch等；
  • Dep就是一个调度器，主要作用是对观察者（Watcher）进行管理，收集观察者和通知观察者目标更新，即当属性值数据发生改变时，通知所有Watcher执行更新。
  • Observer对应下图中的发布器（Publisher）。

# 多对多关系

  • 一个变量对应一个Dep
  • 一个Dep对应一个或多个Watcher（一个变量可以被视图、watch、computed依赖）
  • 一个Watcher对应一个或多个Dep（一个依赖中可能使用了多个变量）

# 订阅

订阅，依赖方订阅数据变化事件，即依赖收集。

Dep收集依赖其的Watcher

在添加响应式的函数defineReactive中，每个数据对象的属性都会创建一个对应的Dep实例dep。在getter方法中（需要用到的时候才进行收集），调用dep.depend()方法，将当前的Watcher（有三种：render函数、computed、watch）添加到内部队列中。

# 观察者获取

dep.depend是通过Dep.target类属性获取观察者的， Dep.target是如何被修改为指向具体的观察者呢？这一过程是在Watcher的getter中添加的。new Watcher(render函数) => Watcher constructor => watcher.get() => get方法中赋值Dep.target = this

Watcher源码示例：

// 1. 创建渲染watcher的时候，把当前的渲染watcher放到Dep.target上
// 2. 调用_render()的时候（构造函数传入的fn），会取值走到属性的getter方法中（模块observer/index defineReactive函数中）

// 观察者模式
// 每个属性对应一个dep，属性就是观察者
// watcher就是观察者
// 属性变化了通知观察者来更新
class Watcher { // 不同的组件有不同的watcher
    constructor(vm, exprOrFn, options, cb) {
        this.vm = vm;
        this.id = id++;
        this.renderWatcher = options; // 标识这是一个渲染watcher
        if (typeof exprOrFn === 'string') {
            this.getter = function () {
                return vm[exprOrFn];
            }
        } else {
            this.getter = exprOrFn; // getter意味着调用这个函数可以发生取值操作
        }
        this.deps = []; // 后续实现计算属性和清理工作（组件卸载）时候使用
        this.depsId = new Set();
        this.lazy = options.lazy;
        this.dirty = this.lazy; // 缓存值

        this.user = options.user;
        this.cb = cb;

        this.value = this.lazy ? undefined: this.get();
    }
    addDep(dep) { // 一个组件对应多个属性，重复的属性不记录
        let id = dep.id;
        if (!this.depsId.has(id)) {
            this.deps.push(dep);
            this.depsId.add(id);
            dep.addWatcher(this);
        }
    }
    evaluate() {
       this.value = this.get(); // 获取到用户函数的返回值，并标识为脏
       this.dirty = false;
    }
    get() {
        Dep.target = this; // 类的静态属性
        pushTarget(this);
        let value = this.getter.call(this.vm); // 会去vm上取值
        // Dep.target = null; // 渲染完毕后就清除掉
        popTarget();
        return value;
    }
    depend() {
        let i = this.deps.length;
        while(i--) {
            this.deps[i].depend(); // 让计算属性watcher也收集渲染watcher
        }
    }
    update() { // 属性每被修改一次，使用属性的组件对应的watcher的update都会被调用一次
       //this.get(); // 重新渲染
       console.log('watcher update');
       if (this.lazy) { // 如果是计算属性
           this.dirty = true; // 依赖的值变化了，就标识计算属性是脏值了
       } else {
           queueWatcher(this);
       }
    }
    run() {
        console.log('watcher run');
        const oldValue = this.value;
        const newValue = this.get();
        if (this.user) {
            this.cb.call(this.vm, newValue, oldValue);
        }
    }
}

# 发布

发布，发布器（Observer）发布事件，调度器（Dep）通知所有观察者（watcher）。

这个事件是什么？添加响应式的属性被修改

defineReactive中的setter被触发，调度器调用dep.notify()方法，遍历观察者列表，通知所有的watcher执行自己的更新逻辑（watcher.update()）。
