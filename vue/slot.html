# 插槽

结合组件原理理解。

# slot实现原理

当子组件vm实例化时，获取到父组件传入的slot标签的内容，存放在vm.$slot中，默认插槽为vm.$slot.default，具名插槽为vm.$slot.xxx，xxx为插槽名，当组件执行渲染函数时候，遇到<slot>标签，使用$slot中的内容进行替换，此时可以为插槽传递数据，若存在数据，则可曾该插槽为作用域插槽。

_t => renderSlot，渲染<slot> _u => resolveScopedSlots

# 具名插槽工作原理

官方示例：

<body>
    <div id="app">
        <base-layout>
            <h1 slot="header">Here might be a page title</h1>
            <p>A paragraph for the main content.</p>
            <p>And another one.</p>
            <p slot="footer">Here's some contact info</p>
        </base-layout>
    </div>
    <script>
        const vm = new Vue({
            el: "#app",
            data: {
                msg: 'outer val'
            },
            components: {
                'base-layout': {
                    data() {
                        return {
                            msg: 'child comp inner'
                        }
                    },
                    template: 
                        `<div class="container">
                            <header>
                                <slot name="header">header 默认文本</slot>
                            </header>
                            <main>
                                <slot></slot>
                            </main>
                            <footer>
                                <slot name="footer"></slot>
                            </footer>
                        </div>`
                }
            }
        })
    </script>
</body>

约定下：父组件指根元素，子组件=>base-layout

# 1. 编译渲染父组件

  • 父组件依次执行_init => $mount => render；
  • 子组件儿子节点render函数（_c('base-layout',[儿子节点render函数])）会先于子组件base-layout的render函数执行。_createElement中，子组件base-layout调用createComponent创建组件的虚拟节点；
  • createComponent中返回VNode倒数第二个选项是组件选项，其中的children就是父组件的孩子节点的VNode。children中如果有插值，会生成并执行对应的_s函数进行取值（从父组件）；

子组件base-layout的render函数：

function anonymous(){
    with(this){
        return _c('div',{attrs:{"id":"app"}},
            [
                _c('base-layout',
                    [
                        _c('h1',{attrs:{"slot":"header"},slot:"header"},[_v("Here might be a page title")]),
                        _v(" "),
                        _c('p',[_v("A paragraph for the main content.")]),
                        _v(""),
                        _c('p'[_v("And another one.")]),
                        _v(" "),
                        _c('p',{attrs:{"slot":"footer"},slot:"footer"},[_v("Here's some contact info")])
                    ]
                )
            ],1)
    }
}

# 作用域问题

父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

createComonent源码：

// src/vdom/create-component
function createComponent() {

    // ...

    const vnode = new VNode(
    // @ts-expect-error
    `vue-component-${Ctor.cid}${name ? `-${name}` : ''}`,
    data,
    undefined,
    undefined,
    undefined,
    context,
    // @ts-expect-error
    { Ctor, propsData, listeners, tag, children }, // 组件选项
    asyncFactory
  )

  return vnode
}

# 2.编译渲染子组件

  • 父组件patch => createElm => 遍历孩子数组createChildren => 遇到子组件base-layout调用createComponent => 子组件init初始化（拿到子组件的构造器Ctor创建实例，然后调用子组件的init我们关心其中的两个：initInternalComponent()、initRender(vm)，然后$mount）
  • initInternalComponent中，将子组件的虚拟节点数组vnode.componentOptions.children赋值给子组件的选项_renderChildren字段
  • initRender中将children虚拟节点按照插槽名称作为key建立索引，存放到vm.$slot、vm.$scopedSlots上，没有给定名称的就创建一个key为default
function initInternalComponent() {
    // core/instance/render
    // ...
    const vnodeComponentOptions = parentVnode.componentOptions!
    opts.propsData = vnodeComponentOptions.propsData
    opts._parentListeners = vnodeComponentOptions.listeners
    opts._renderChildren = vnodeComponentOptions.children
}

// core/instance/render
function initRender() {
    const options = vm.$options
    const parentVnode = (vm.$vnode = options._parentVnode!) // the placeholder node in parent tree
    const renderContext = parentVnode && (parentVnode.context as Component)
    vm.$slots = resolveSlots(options._renderChildren, renderContext)
}

# 3.<slot>编译渲染

  • 子组件init初始化完成，调用$mount，编译模板遇到<slot>标签在解析生成render函数阶段（codegen）会由genSlot函数处理，生成_t函数；
  • _t函数对应renderSlot在render函数执行阶段调用，生成slot的虚拟节点；
  • renderSlot中，从vm.$slot这个映射表上获取虚拟节点，具名插槽的name就是其名称，默认插槽的name就是default；（v2.6.0之后的版本是从vm.$scopedSlots获取）；
  • 然后patch创建子组件真实节点。

子组件模板对应的render函数：

function anonymous(){
	with(this){
		return _c('div',{staticClass:"container"},
			[
				_c('header',[_t("header",function(){return[_v("header默认文本")]})],2),
				_v(" "),
				_c('main',[_t("default")],2),
				_v(" "),
				_c('footer',[_t("footer")],2)
			]
		)
	}
}

源码参考：

// core/instance/render-helpers/renderSlot
export function renderSlot(
  name: string,
  fallbackRender: ((() => Array<VNode>) | Array<VNode>) | null,
  props: Record<string, any> | null,
  bindObject: object | null
): Array<VNode> | null {
  const scopedSlotFn = this.$scopedSlots[name]
  let nodes
  if (scopedSlotFn) {
    // scoped slot
    props = props || {}
    if (bindObject) {
      if (__DEV__ && !isObject(bindObject)) {
        warn('slot v-bind without argument expects an Object', this)
      }
      props = extend(extend({}, bindObject), props)
    }
    nodes =
      scopedSlotFn(props) ||
      (isFunction(fallbackRender) ? fallbackRender() : fallbackRender)
  } else {
    nodes =
      this.$slots[name] ||
      (isFunction(fallbackRender) ? fallbackRender() : fallbackRender)
  }

  const target = props && props.slot
  if (target) {
    return this.$createElement('template', { slot: target }, nodes)
  } else {
    return nodes
  }
}

# 作用域插槽工作原理

旧版API示例：

<slot-example>
  <template slot="default" slot-scope="slotProps">
    {{ slotProps.msg }}
  </template>
</slot-example>

新版API示例：

<!--子组件-->
<span>
  <slot v-bind:user="user">
    {{ user.lastName }}
  </slot>
</span>
<!--子组件-->
<!--父组件-->
<current-user>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</current-user>
<!--父组件-->

<body>

            <template v-slot:address="otherProps">

# 模板编译阶段

  • 模板解析parseHTML阶段，closeElement函数中，processSlotContent函数被调用处理<template>标签，将v-slot、slot-scope属性的值提取放到AST节点（<template>）的slotScope属性上；
  • 解析继续执行，将<template>节点的slotScope属性放到父组件的scopedSlots属性上
  • codegen生成render函数阶段，分别调用genScopedSlots、genScopedSlot，相比普通插槽父组件，作用域插槽父组件生成函数多了一个属性scopedSlots

父组件render函数：

function anonymous(){
    with(this) {
        return _c('div', {attrs:{"id":"app"}},
        [_c('current-user',
        {
            scopedSlots:
            _u(
                [
                    {
                        key:"user",
                        fn: function(slotProps){
                            return [_v("\n"+_s(slotProps.userfirstName)+"\n ")]
                        }
                    }, 
                    {
                        key: "address",
                        fn: function(otherProps) {
                            return [_v("\n "+_s(otherProps.info)+"\n ")]
                        }
                    }
                ]
            )
        })], 1)
    }
}

# render函数执行阶段

  • _u对应的resolveScopedSlots函数执行，将数组参数中的每个元素转换成slot名称和对应函数的键值对，并赋给scopedSlots
  • 那么生成的父组件虚拟节点上就有了这个函数映射，vnode.componentOptions.scopedSlots

_u执行后得到：

_c('current-user',
        {
            scopedSlots: {
                "user": function(slotProps){
                    return [_v("\n"+_s(slotProps.userfirstName)+"\n ")]
                },
                "address": function(otherProps) {
                    return [_v("\n "+_s(otherProps.info)+"\n ")]
                }
            }
        })

# 2.<slot>处理

gencode阶段，<slot>标签被genSlot处理，attrs和v-bind，生成代码会有绑定映射

function anonymous() {
    with(this){
        return _c('span',
        [
            _t("user",function(){return[_v(_s(user.lastName))]},{"user":user}),
            _v(" "),
            _t("address",function(){return [_v(_s(city))]},{"info":city})],2)
    }
}
  • render函数执行阶段，_t对应的renderSlot函数执行
  • $scopedSlots中，属性上获取对应的渲染函数并执行获得VNode（因为执行上下文在子组件，所以实现了作用域切换）
  • $scopedSlots是在组件init过程中initRender函数中从组件vnode.data.scopedSlots上获取
  • 最后再patch生成真实节点
function renderSlot() {
    const scopedSlotFn = this.$scopedSlots[name]
    nodes =
      scopedSlotFn(props) ||
      (isFunction(fallbackRender) ? fallbackRender() : fallbackRender)
    return nodes  
}

# 对比区别

  • 编译阶段：生成的render函数不同
    • 普通插槽，插槽儿子节点被编译成，普通节点render函数（_c、_v）放入儿子数组；
    • 作用域插槽，插槽儿子节点会被编译成函数，放到组件的scopedSlots属性上
  • render阶段：
    • 普通插槽，插槽子节点vnode在外层父组件render阶段就已生成，然后传给组件，生成vnode时候直接复用
    • 作用域插槽，生成vnode阶段调用子节点对应的渲染函数生成儿子的vnode
