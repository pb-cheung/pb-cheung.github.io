# 数字

# 规范

JavaScript中的数字均为双精度浮点（Double-precision floating-point format (opens new window)）类型。

遵循的标准规范是：IEEE 754

这样的存储结构优点是可以归一化处理整数和小数，节省存储空间。

# 科学计数法

规范中，数字是以2进制数科学计数法的形式存储。

# 10进制的科学计数法

一个10进制数，可以被记为，一个实数a和10的n次幂的乘积的形式：a * 10ⁿ,

  • a——尾数，1 <= |a| < 10
  • n——指数

优点： 可以很容易的看出一个数的数量级（极大数）、精确度（极小数）和数值（尾数）。 极大数和极小数不用写很多0。

# 2进制的科学计数法

计算机中使用2进制。2进制的科学计数法为：a * 2ⁿ。

a是由0、1组成的二进制数。1 < a < 2

例如：

6被存储为：1.10 * 2² 15被存储为：1.111 * 2³

# 存储结构

  • 符号位（Signal）：1位，0-负数，1-正数
  • 指数位（Exponent [ɪkˈspəʊnənt]）：11位，表示指数
  • 尾数（Mantissa [mæn'tɪsə]）：52位，尾数

# 一、偏差指数

指数被存储为无符号值，在解释的时候需要"减去偏差"就能变成有符号的指数了。

偏差的值是 2ᴷ⁻¹-1，其中 k 是指数的位数。比如：

  • 单精度：指数有 8 位，指数偏差的值是 2⁸⁻¹-1 = 127
  • 双精度：指数有 11 位，指数偏差的值是 2¹¹⁻¹-1 = 1023
  • 四精度：指数有 15 位，指数偏差的值是 2¹⁵⁻¹-1 = 16383

# 三、十进制转二进制

# 整数部分——阶除法

除2取余，逆序排列

  1. 整数部分除以2，记录余数（0 or 1）
  2. 所有余数从下到上，倒着记录即为结果

# 小数部分

乘2取整，顺序排列

  1. 小数部分乘以2，记录乘积的整数部分
  2. 再用上一步的积的小数部分乘以2
  3. 重复1.2.操作，直到积的余数为0，或者达到需要的精度
  4. 然后将记录的值从上到下排列

# 面试题

# 1. 0.1 + 0.2 === 0.3 嘛？为什么？

  • 不等于
  • 原因：

这是因为在计算机中使用IEEE 754标准来表示浮点数（包括小数），而该标准无法精确地表示所有的十进制小数。

在IEEE 754中，浮点数被表示为二进制的科学计数法，其中小数部分被存储为二进制分数。0.1 和 0.2 这两个十进制数转换成二进制时会无限循环，类似于十进制中 1/3 无限循环一样。这导致在存储这些值时，会有一些舍入误差。

0.1 + 0.2 = 0.30000000000000004

# Number() 的存储空间是多大？如果后台发送了一个超过最大自己的数字怎么办

  • 64位。
  • 超大数字处理
    • 可以使用字符串（如果只展示的话）
    • 使用BigInt，BigInt(xxx)，或者大数末尾加n
    • 使用bignumber.js (opens new window)

BigInt 使用了一种称为 "arbitrary-precision arithmetic"（任意精度算术） 的技术。它可以动态地分配足够的内存来存储整数，并根据实际需要调整内存空间大小，只受计算机内存的限制。
